# EKS Engineering Playbook 소개

**EKS Engineering Playbook**에 오신 것을 환영합니다! 이 플레이북은 Amazon EKS(Elastic Kubernetes Service) 관련 아키텍처 딥다이브 문서들을 체계적으로 정리한 종합 가이드입니다.

## 🎯 목적

이 플레이북은 다음과 같은 목적으로 만들어졌습니다:

- EKS 관련 복잡한 기술 주제들을 5개 핵심 도메인으로 체계화
- 실무에서 바로 적용 가능한 아키텍처 패턴과 모범 사례 제공
- 단계별 구현 가이드와 실제 코드 예제 포함

## 📚 기술 도메인

### 🚀 [Performance & Networking](./performance-networking/)
- EKS DNS 성능 최적화
- Cilium ENI 모드 마스터하기
- CoreDNS 모니터링 및 최적화

### 👁️ [Observability & Monitoring](./observability-monitoring/)
- Hubble을 통한 EKS 관찰 가능성
- AI/ML 워크로드 모니터링
- Langfuse와 Amazon EKS 통합

### 🤖 [GenAI & AI/ML](./genai-aiml/)
- 프로덕션 레디 GenAI 구축
- GPU 효율성 극대화 전략
- EKS 기반 GenAI 플랫폼 구축

### 🌐 [Hybrid & Multi-Cloud](./hybrid-multicloud/)
- EKS를 클라우드 너머로 확장
- 하이브리드 워크로드 스케일링
- EKS 하이브리드 노드 설정

### 🔒 [Security & Compliance](./security-compliance/)
- ROSA 네트워크 보안 컴플라이언스
- 보안 아키텍처 모범 사례
- 컴플라이언스 구현 가이드

## 🚀 시작하기

1. 관심 있는 기술 도메인을 선택하세요
2. 각 도메인의 문서들을 순서대로 읽어보세요
3. 제공된 코드 예제와 설정을 실제 환경에 적용해보세요

## 📖 문서 구조

각 문서는 다음과 같은 구조로 구성되어 있습니다:

- **개요**: 주제에 대한 간단한 소개
- **아키텍처**: 시스템 구조와 컴포넌트 설명
- **구현 가이드**: 단계별 설정 및 구현 방법
- **코드 예제**: 실제 적용 가능한 YAML, 스크립트 등
- **모니터링**: 성능 측정 및 문제 해결 방법
- **모범 사례**: 프로덕션 환경에서의 권장사항
