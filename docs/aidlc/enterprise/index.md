# AIDLC 엔터프라이즈 도입

> **읽는 시간**: 약 2분

AIDLC 방법론을 실제 엔터프라이즈 환경에 도입하려면, 기술 방법론 이상의 **조직 변환, 비용 정량화, 거버넌스 체계**가 필요합니다. 특히 워터폴 기반 SI 시장, 고정가 입찰(RFP), 다층 거버넌스, 데이터 레지던시 등 엔터프라이즈 고유의 과제를 체계적으로 다룹니다.

## 핵심 과제

엔터프라이즈 AIDLC 도입에서 가장 자주 직면하는 과제들입니다:

- **프로세스 전환**: 워터폴에서 AIDLC 하이브리드로 어떻게 전환할 것인가?
- **비용 정당화**: AIDLC 도입이 프로젝트 비용을 얼마나 절감하는가?
- **역할 변화**: PM, 아키텍트, 개발자의 역할이 어떻게 바뀌는가?
- **품질 거버넌스**: AI 생성 코드의 품질을 전사적으로 어떻게 관리하는가?
- **복잡도 관리**: 대규모 MSA 프로젝트에도 AIDLC를 적용할 수 있는가?

## 방법론과의 연결

엔터프라이즈 도입 트랙은 [방법론](/docs/aidlc/methodology)의 핵심 개념을 조직·비즈니스 맥락에서 재해석합니다:

- **온톨로지** → [비용 효과](./cost-estimation.md): 에러율 감소 ROI, 리뷰 공수 절감
- **하네스** → [역할 재정의](./role-composition.md): 하네스 엔지니어 신규 역할 / [비용 효과](./cost-estimation.md): 하네스 부재 비용
- **오픈 웨이트 모델** → [거버넌스](./governance-framework.md): 데이터 주권 대응 / [비용 효과](./cost-estimation.md): TCO 비교
